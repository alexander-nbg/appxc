# Drafts

```{page-status} usable
```

[Feature](/user/features/_features) and [module](/user/modules/_modules) pages
are placed here before or during implementation to keep the user area clean. They should
be present before implementation since module context and user perspective should be
clarified before implementation. Latest, when modules become usable, the sections are
moved in the user area.

Both page layers may include the analysis of alternatives. If reasonable, they may be
concluded into ticket comments, code comments and refined into user documentation.
However, keeping draft pages in addition to corresponding user pages to keep the
analysis of alternatives is perfectly fine. Having pages in both areas will also occur
if existing features or modules are extended.
